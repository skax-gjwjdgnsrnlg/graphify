<objective>
Fix the broken DRAFT→PAPER_LIVE activation chain (the phase's most important control). Currently `PaperLifecycleService.promote()` only flips `status='PAPER_LIVE'` and never populates `paper_live_symbols`, so `LiveDataScheduler` Guard 3 (`activeSymbolsUnion()`) finds an empty table and skips every tick — no ingestion, no evaluation, no trades. Additionally `LiveEvaluationService.resolveSymbols()` reads symbols from `rule.definition.universe` while the scheduler ingests from `paper_live_symbols`, so the two halves of the loop are derived from different sources.

Purpose: Make promote actually start live paper trading end-to-end, with a single canonical symbol source.
Output: A promoted rule populates `paper_live_symbols`, the scheduler tick runs, and `LiveEvaluationService` evaluates exactly the assigned symbols.

**Symbol-source reconciliation decision (EXPLICIT):** `paper_live_symbols` is the SINGLE canonical source for both ingestion and evaluation. promote() resolves the rule's symbols once (mirroring `BacktestService.resolveInitialSymbols`) and writes them to `paper_live_symbols`; `LiveEvaluationService.resolveSymbols()` is changed to READ from `paper_live_symbols` (via `PaperLiveSymbolRepository.findByRuleId`) instead of parsing `rule.definition.universe`. This guarantees ingested symbols == evaluated symbols and enables `volume_top_n` live rules (the gap flagged in 03-VERIFICATION "notable").
</objective>
