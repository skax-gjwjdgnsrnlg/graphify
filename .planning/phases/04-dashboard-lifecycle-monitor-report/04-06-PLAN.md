<objective>
Make the rule-lifecycle promote UI reachable in default PAPER mode. Today `TradingRulesPage` (the only page with promote/pause/resume/copy buttons) is routed at `/trading/rules` under `ModeGuard mode="LIVE"` and listed only in the LIVE sidebar. In default PAPER mode the route redirects to `/trading/paper/dashboard`, and the PAPER nav's `PaperRulesPage` (a CRUD editor) offers only DRAFT/ACTIVE/PAUSED with no promote. So a normal PAPER-mode user has no path to "PAPER_LIVE 승격".

Purpose: Give PAPER-mode users a reachable place to promote a backtested rule to PAPER_LIVE.
Output: A new PAPER-mode route + sidebar entry that renders the existing fully-implemented `TradingRulesPage` lifecycle list. LIVE behavior untouched.

Decision: Reuse the existing `TradingRulesPage` component (no rewrite — it already renders badges + promote/pause/resume/copy and is wired to `paperApi`). Add a new PAPER-mode route `/trading/paper/rules-lifecycle` wrapped in `ModeGuard mode="PAPER"` and a PAPER sidebar entry. Leave the existing LIVE `/trading/rules` route and LIVE sidebar entry exactly as they are.
</objective>
