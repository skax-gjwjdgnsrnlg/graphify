## Implementation Details

### YahooCumulativeVolumeAdapter

**File:** `backend/src/main/java/com/graphify/market/volume/YahooCumulativeVolumeAdapter.java`

- `@Component("yahooCumulativeVolumeAdapter")` — explicit bean name for `@Qualifier` injection in Plan 03
- Implements `VolumeRankingProvider.topVolume(market, date, topN, excludeEtf)`
- Computes KST day boundaries (`dayStart`/`dayEnd` as `Instant`) from `LocalDate` parameter
- Calls `MarketBarIntradayRepository.findCumulativeVolumeByMarketAndDate(market, dayStart, dayEnd, PageRequest.of(0, topN))`
- Exception: `log.warn + stale cache/empty list` (no propagation)
- `excludeEtf` parameter: JPQL always filters `COMMON_STOCK` — parameter is semantically redundant but accepted per port contract

### findCumulativeVolumeByMarketAndDate JPQL

**File:** `backend/src/main/java/com/graphify/market/MarketBarIntradayRepository.java`

```
SELECT m.symbol
FROM MarketBarIntraday m
JOIN com.graphify.company.Company c ON c.ticker = m.symbol
WHERE c.market = :market
  AND c.instrumentType = 'COMMON_STOCK'
  AND m.interval = '5m'
  AND m.ts >= :dayStart
  AND m.ts < :dayEnd
  AND m.volume IS NOT NULL
GROUP BY m.symbol
ORDER BY SUM(m.volume) DESC
```

- Pageable drives `LIMIT topN` at DB level (efficient — no post-filter)
- `instrument_type = 'COMMON_STOCK'` enforces V36 ETF/ETN/우선주/SPAC exclusion (DATA-06-SC2)

### Cache Design (1-min TTL)

Per RESEARCH §캐시 설계 (IntradayBarCacheService 패턴):

```java
private volatile List<String> cache = List.of();
private volatile Instant cacheTs = Instant.EPOCH;
static final Duration CACHE_TTL = Duration.ofMinutes(1);
```

- Cache hit: `Instant.now().isBefore(cacheTs.plus(CACHE_TTL)) && !cache.isEmpty()`
- Empty fetch → cache NOT updated (RESEARCH Pitfall 5 — prevents stale-empty poisoning)
- Exception → stale cache returned (or empty if never populated)
