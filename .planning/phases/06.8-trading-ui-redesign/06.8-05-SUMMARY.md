### Implementation Notes (not deviations)

**1. Textarea retained for DDS chat input**
- **Found during:** Task 1
- **Reason:** `TradeInput` renders `<input>` (HTMLInputElement), not `<textarea>`. Auto-height behavior (`onInput` style mutation + `maxHeight: 8rem`) requires a `<textarea>` element. Replaced cream classes with trade tokens inline — functionally equivalent to using `TradeInput` for a single-line field. Plan explicitly allowed "(or themed textarea)".

**2. `aria-label` omitted from send TradeButton**
- **Found during:** Task 1
- **Reason:** `TradeButtonProps` does not extend `React.ButtonHTMLAttributes<HTMLButtonElement>`. Passing `aria-label` would cause a TypeScript build error. Removed; icon context is self-evident within the chat form.

**3. Leak guard CandleSection.tsx — pre-existing, out of scope**
- `src/components/backtest/CandleSection.tsx` contains `trade-*` tokens from Phase 06.6 (documented in STATE.md Phase 06.6 decisions: "CandleSection 4 dark inline states — light cream tokens incompatible with dark gray-900 theme"). Plan 05 introduced zero new leaks.
